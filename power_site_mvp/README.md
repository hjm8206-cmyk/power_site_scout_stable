## 도로 수동보정 최신 기준
도로는 500m까지 자동 검색합니다. 왼쪽 메뉴의 수동보정은 `10m`, `6m`, `4m`만 제공합니다. 농로 또는 비포장로는 개발 진입도로로 보지 않아 수동 선택지에서 제외하고, 자동판정에서 농로추정으로 잡히더라도 도로 점수는 0점으로 처리합니다. 수동보정 점수는 `10m=20점`, `6m=15점`, `4m=5점`입니다. 내부적으로는 공식도로, 도로명도로, 도시계획도로, 실폭도로, 농로추정, 임도추정, 미확인도로, 접도불명확을 구분하고 최대 3필지 경유 접도 가능성을 분석합니다.

진입도로가 실제로는 있으나 협소하거나 장거리라 공사차량 진입이 어렵다고 확인되는 경우 별도 체크로 표시합니다. 이 경우 도로 기본점수는 0점으로 처리하지만, 맹지 또는 500m 내 도로 후보 없음 감점은 적용하지 않습니다.

## 용도지역·경사도 최신 기준

용도지역 점수는 복잡한 수동목록 대신 5개 큰 그룹으로 단순 분류합니다. 계획관리지역·공업지역·준공업지역은 고점수 구간으로 보고, 보전관리지역과 생산관리지역은 같은 점수대의 검토 가능 구간으로 처리합니다. 성장관리계획구역은 기본 수동확인 항목으로 노출하지 않고, 용도지역·지구 데이터에서 명확히 검색될 때만 결과와 점수에 반영합니다. 보전관리·생산관리라는 이유만으로 낮은 점수를 주지 않으며, 전력축·도로·민가·정책입지 조건이 좋으면 17~18점까지 반영할 수 있습니다. 농림지역은 낮은 점수와 별도 감점을 적용하되 전략검토 여지를 남기고, 개발제한구역은 최악 구간으로 처리합니다.

경사도는 일반지도나 위성사진 화면을 기준으로 판단하지 않고 DEM 또는 등고선 기반 자동계산 구조를 사용합니다. 하이브리드 지도는 시각 검증용 기본 화면이며, 가능한 경우 지형/등고선 오버레이를 함께 표시합니다. 경사도 자동조회 실패는 부지 경사가 높다는 의미가 아닙니다. DEM/등고선 데이터 미연동 또는 조회 실패로 인한 확인 필요 상태이므로 자동조회 실패만으로 감점이나 최종점수 상한을 적용하지 않습니다. 경사도 감점은 자동계산값이 있거나 사용자가 수동으로 경사도 등급을 선택한 경우에만 적용합니다. 경사도 미확인 상태에서는 경사도 항목을 제외한 95점 기준 점수를 100점으로 환산하고, 등급에 조건부 표시를 붙입니다.

현재 MVP에서는 지도 백화 현상을 막기 위해 Kakao 지형/등고선 오버레이와 DEM/등고선 자동 경사도 계산을 비활성화했습니다. 경사도는 고급설정의 수동 경사도 선택값만 점수에 반영하며, 수동 선택이 없으면 경사도 감점과 최종점수 상한을 적용하지 않습니다.

## 민감시설 자동탐지 기준

민감시설은 사용자가 수동 체크하지 않고 Kakao/VWorld 데이터 기반 자동탐지 결과만 사용합니다. 어린이집, 유치원, 학교, 병원, 요양시설, 어린이공원은 민감시설로 분류하고, 아파트·공동주택단지는 별도 주거단지 리스크로 분류합니다.

감점 거리는 선택 부지 polygon 경계 기준 최단거리를 우선 사용합니다. 선택 부지 polygon이 없으면 메인 필지 경계, 그것도 없으면 주소 기준점 거리를 임시로 사용합니다. 500m 초과 1km 이내 탐지 결과는 참고값으로만 표시하며 자동감점이나 최종점수 상한을 적용하지 않습니다.

## 최소 사업구역 면적 기준

본 앱은 부지제공, 신설변전소, 대용량 수전형 데이터센터 후보지를 평가하는 내부 스카우트 도구입니다. 데이터센터 입지 평가는 단일 필지가 아니라 사용자가 선택한 전체 필지 묶음, 즉 사업구역 기준으로 계산합니다.

선택된 개발부지 필지군의 최종 합산 면적이 10,000평 미만이면 종합점수를 산정하지 않습니다. 이 경우 화면과 리포트에는 “대용량 수전형 데이터센터 검토 불가”, “선택한 필지 총합 면적이 10,000평 미만입니다”, “부지제공·신설변전소·대용량 수전 기준상 최소 사업구역 면적 10,000평 이상이 필요합니다”, “인접 필지를 추가 선택한 후 다시 평가하세요”를 표시합니다.

10,000평 이상일 때만 전력, 인허가, 민원, 정책입지, 기반시설 종합점수를 계산합니다. 10,000평 이상이라고 별도 가점을 주지는 않으며, 면적은 종합평가 진입을 위한 최소요건으로만 사용합니다.

자동탐지가 실패한 경우에는 감점하지 않고 “민감시설 자동조회 실패 / 수동확인 필요”로만 표시합니다. 민감시설 또는 주거단지 감점이 적용되는 경우에는 감점, 거리 기준, 탐지 소스, 신뢰도를 결과 패널과 리포트에 함께 표시하며 final_score에 반드시 반영합니다.

## 전력축 인접성 기준

전력축 인접성은 주소 기준점이 아니라 선택 부지군 polygon 경계 기준으로 계산합니다. 송전탑이나 송전선 후보축이 부지 내부 또는 경계부에 있을 때 가장 높은 점수를 부여하며, 50m 이내는 우수, 50~150m는 검토 가능, 150m를 초과하면 강한 전력축 인접성으로 보지 않고 참고값에 가깝게 처리합니다.

송전탑 또는 송전선 후보축이 부지 내부에 있을 경우 전력축 인접성은 높지만, 선하지·안전거리·점용·이설·한전 협의·민원 리스크를 별도로 확인해야 합니다.

## 수동마킹 도로 접도 반영

지도에서 도로 마킹 모드를 켜고 도로 위를 2점 이상 클릭하면 수동도로 polyline을 생성합니다. 수동도로에는 `10m`, `6m`, `4m`, `미확인/없음` 폭원을 지정할 수 있으며, 수동마킹 도로가 있으면 자동조회 도로보다 우선해 도로점수에 반영합니다.

수동도로는 Shapely 기반 거리 계산으로 메인 필지, 편입 후보 필지, 도로 연결 후보 필지와의 접촉 여부를 판단합니다. 기본 허용오차는 5m입니다. 메인 필지에 직접 닿으면 직접 접도, 편입 후보를 통해 닿으면 편입 후보 포함 직접 접도, 도로 연결 후보 필지를 경유하면 최대 3필지 경유 접도로 분류합니다. 도로 연결 후보 필지는 access parcel로만 처리하며 개발면적에는 자동 합산하지 않습니다.

수동도로 점수는 폭원 기본점수에서 접도방식 감점을 빼서 계산합니다. 기준은 `10m=20점`, `6m=15점`, `4m=5점`, `미확인/없음=0점`이며, 직접 접도는 감점 없음, 편입 후보 포함 접도는 -2점, 1필지 경유는 -3점, 2필지 경유는 -5점, 3필지 경유는 -7점입니다. 수동도로가 선택 부지군과 전혀 접도하지 않으면 도로점수에 반영하지 않고 0점으로 표시합니다.

```text
APP_LOGIN_PASSWORD=팀 로그인 비밀번호
```

지도 표시에는 Kakao JavaScript 키도 필요합니다. 이 키는 브라우저에서 쓰이는 공개형 키이므로 `KAKAO_JS_KEY`로 등록하고, 카카오 개발자 콘솔에서 Vercel 배포 도메인을 JavaScript 플랫폼 도메인으로 제한합니다.

```text
KAKAO_JS_KEY=카카오 JavaScript 키
APP_SESSION_SECRET=긴 랜덤 문자열(선택, 권장)
VWORLD_DOMAIN=https://배포도메인.vercel.app
```

`KAKAO_REST_API_KEY`와 `VWORLD_API_KEY`는 서버 코드에서만 읽습니다. 주소검색은 브라우저에서 카카오 Local REST API를 직접 호출하지 않고 `/api/analyze` 서버 API route를 통해 처리합니다.

### 로그인 보호

`APP_LOGIN_ID`, `APP_LOGIN_PASSWORD`가 설정되어 있으면 전체 평가툴은 로그인 후에만 접근할 수 있습니다. 로그인 전에는 화면 렌더링과 `/api/*` 호출이 차단되며, API는 `401 로그인이 필요합니다`를 반환합니다. Vercel 환경에서는 로그인 환경변수가 없으면 앱 대신 설정 오류 화면을 보여줍니다.

### 배포 순서

1. GitHub 저장소에 `power_site_mvp` 폴더를 푸시합니다.
2. Vercel에서 New Project를 만들고 Root Directory를 `power_site_mvp`로 설정합니다.
3. Environment Variables에 위 필수 키와 로그인 정보를 등록합니다.
4. Build Command는 `npm run build`를 사용합니다.
5. 배포 후 카카오 개발자 콘솔 JavaScript 플랫폼 도메인과 VWorld 서비스 URL에 Vercel 도메인을 등록합니다.
6. 배포 URL에서 로그인 후 주소검색, 필지추가, 감점 체크, 정책점수, 총점표, Markdown 리포트를 확인합니다.

### 로컬 빌드 확인

```bash
npm run build
```

빌드 스크립트는 배포에 필요한 파일 존재 여부와 `static/app.js` 문법을 확인합니다. Python 문법 확인은 아래 명령으로 별도 실행할 수 있습니다.

```bash
python -m compileall app
```

### 보안 주의

- `.env`, `reports`, `private_data`, `backups`, `.vercel`, `node_modules`는 Git에 커밋하지 않습니다.
- 실제 주소, 지번, 후보지 정보, API 키는 저장소에 올리지 않습니다.
- Kakao REST 키는 브라우저에 노출하지 않습니다.
- Kakao JavaScript 키는 지도 표시용 공개 키이므로 반드시 도메인 제한을 걸어 사용합니다.
